# LiveOps Building Blocks

LiveOps Unity Building Blocks use Unity Gaming Services (UGS) to add player sign-in options, achievements, and a leaderboard. You can add a LiveOps Building Block at any point in your project’s development.

| **Topic** | **Description** |
| --- | --- |
| [LiveOps Building Blocks prerequisites](building-blocks-liveops-prerequisites.html) | Learn how to set up your project to work with the [Unity Services](UnityServices.html) that the LiveOps Building Blocks use. |
| [Player Accounts Building Block](building-blocks-liveops-player-accounts.html) | Add a sign-in workflow and **scripts**A piece of code that allows you to create your own Components, trigger game events, modify Component properties over time and respond to user input in any way you like. [More info](creating-scripts.html) See in [Glossary](Glossary.html#Scripts) to control player data to your **scene**A Scene contains the environments and menus of your game. Think of each unique Scene file as a unique level. In each Scene, you place your environments, obstacles, and decorations, essentially designing and building your game in pieces. [More info](CreatingScenes.html) See in [Glossary](Glossary.html#Scene). |
| [Achievements Building Block](building-blocks-liveops-achievements.html) | Add an achievement system to your scene and learn how to use them. |
| [Leaderboards Building Block](building-blocks-liveops-leaderboard.html) | Add a leaderboard to your scene to display and rank user scores. |
